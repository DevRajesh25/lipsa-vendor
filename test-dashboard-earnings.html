Dashboard Earnings Debug

Instructions

  1. Open browser console (F12)
  2. Go to vendor dashboard page
  3. Look for console logs starting with "==="
  4. Check the following:

What to Check in Console

=== Calculating stats for vendor: [vendorId]
Total products: [number]
Orders snapshot size: [number]
Order data: {
  id: "...",
  vendorId: "...",
  vendors: ["..."],
  vendorEarnings: 270,  ← Should be a number
  vendorAmount: ...,
  paymentStatus: "..."
}
Total orders fetched: [number]
Processing [number] orders for vendor [vendorId]
Order [id]: Single-vendor earnings = 270
Order [id]: paymentStatus = ..., vendorEarning = 270
Total earnings calculated: 270
Pending payout: 270

Common Issues

  • Orders snapshot size: 0 - Vendor ID mismatch or no orders in database
  • vendorEarnings: undefined - Field not set in Firestore
  • Total earnings: 0 - Check if vendorId matches or paymentStatus filter

Expected Flow

1. Query orders where vendors array contains vendorId
2. For each order:
   - Check if vendorEarnings is a number
   - If yes and vendorId matches, use that value
   - Add to totalEarnings
3. Display totalEarnings on dashboard
